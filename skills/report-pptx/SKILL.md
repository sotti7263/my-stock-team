# report-pptx

`reports/{종목명}.md` 를 읽어 **디자인된 PPTX 리포트**를 `reports/{종목명}.pptx` 로 생성하는 스킬입니다.

## 실행

플러그인 번들 스크립트는 `${CLAUDE_PLUGIN_ROOT}` 로 참조합니다.

```bash
# PPTX 생성 → reports/{종목명}.pptx
python "${CLAUDE_PLUGIN_ROOT}/skills/report-pptx/build_pptx.py" 삼성물산
# PDF 생성 → reports/{종목명}.pdf (PowerPoint/LibreOffice 불필요)
python "${CLAUDE_PLUGIN_ROOT}/skills/report-pptx/build_pdf.py" 삼성물산
# 경로 직접 지정도 가능
python "${CLAUDE_PLUGIN_ROOT}/skills/report-pptx/build_pptx.py" reports/삼성물산.md
```

의존성: `python-pptx`, `matplotlib`(PDF용). 미설치 시 `pip install python-pptx matplotlib`. 한글 폰트 `맑은 고딕`은 Windows 기본 탑재이며, 없으면 기본 폰트로 자동 대체합니다(PDF).

## 슬라이드 순서 (항상 이 순서로 고정)
1. **표지** — 종목명 · 작성일
2. **종목 개요**
3. **재무 요약** — 표(최근 3개년)
4. **가격/추세** — 차트
5. **뉴스·심리**
6. **리스크**
7. **한 줄 종합**

## 디자인 규칙 (스크립트에 내장)
- 포인트색 **KB 옐로우 `#FFBC00`** + 본문 그레이/화이트, 차분한 금융 리포트 톤
- 한글 폰트는 **맑은 고딕** 하나로 고정 (글자 깨짐 방지)
- 표는 슬라이드 밖으로 넘치지 않도록 **행 수·폰트 자동 제한** (재무는 3개년, 그 외 6~8행 상한, 초과분은 생략 표기)
- 매수/매도·목표가 단정 표현은 **가드레일 린트**로 경고 (감지 시 콘솔에 라인 표시)

## 입력 `.md` 작성 규칙 (계약)

스크립트는 **`.md`에 적힌 내용만 렌더**합니다 — 숫자를 새로 만들지 않으므로, 모든 수치와 출처는 `.md`에 먼저 적혀 있어야 합니다.

- 첫 줄에 제목: `# {종목명}` (예: `# 삼성물산`)
- 어디든 한 줄: `작성일: YYYY-MM-DD`
- 아래 **H2 섹션**을 둡니다(제목 키워드로 자동 인식, 순서 무관):

  | 섹션 제목(예) | 인식 키워드 | 권장 내용 |
  |---|---|---|
  | `## 종목 개요` | 개요 | 불릿 또는 항목 표 |
  | `## 재무 요약` | 재무·실적 | **마크다운 표**(연도×지표) + 코멘트 불릿 |
  | `## 가격/추세` | 가격·추세·차트 | ` ```chart ` 블록 + 코멘트 불릿 |
  | `## 뉴스·심리` | 뉴스·심리 | 불릿(이슈별 출처·날짜 포함) |
  | `## 리스크` | 리스크·위험 | 불릿 또는 표 |
  | `## 한 줄 종합` | 종합·결론 | 첫 불릿이 강조 박스로, 나머지는 보조 불릿 |

### 수치 출처 표기
- 표 아래에 `출처: ...` 로 시작하는 줄을 두면 **표 캡션**으로 렌더됩니다. (표 셀마다 출처를 반복하지 않아 오버플로를 막습니다.)
- 불릿 안 수치는 문장에 `(출처: …, 기준일 …)` 형태로 직접 적습니다.
- **출처를 댈 수 없는 숫자는 `.md`에 싣지 않습니다.**

### 가격 차트 데이터
가격/추세 섹션에 아래 펜스 블록을 넣으면 **꺾은선 차트**로 그립니다(없으면 가격 요약표로 대체):

````
```chart
date,close
2025-12-11,245500
2026-01-15,300000
...
```
````

### 가드레일
- "사야 합니다 / 팔아야 합니다 / 목표가 / 적정주가" 등 단정 표현 금지 — **판단 근거까지만** 서술합니다.
- 표지 외 모든 슬라이드 하단에 `본 자료는 학습용 분석이며 투자 권유가 아닙니다.` 가 자동 삽입됩니다.

## 예시 입력
입력 `.md` 는 위 H2 섹션 규칙을 따르는 종목 리서치 파일입니다. `/stock-research <종목명>` 커맨드로 생성된 `reports/{종목명}.md` 를 그대로 입력으로 사용할 수 있습니다.
